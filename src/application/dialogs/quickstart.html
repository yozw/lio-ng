Getting started tour

Welcome to this Linear Optimization Solver. In this Getting Started dialog, we will briefly guide you through this app.

On the left of the screen, there are a number of tabs: Documentation, Model, and Solution (which itself has a number of sub-tabs). We will go through each of them in the next few steps.

Click 'Next' to continue.

Many built-in models in the app have description of the problem that the model solves, and the modeling approach. The Documentation tab shows this description for the currently loaded model.

Click 'Next' to continue.

The Model tab shows the GMPL code for the model. This is computer code that defines the mathematical in a suitable way for the solver. In the code, the variables, the objective function, and the constraints are declared.

Click 'Next' to continue.

The Solution tab shows the results from computations with the model. Clicking the green "Solve model" button in the bottom left corner of the screen starts the solution process of the current model:

Since solving the model can take some time, the solution process runs in the background. Once the solution process is finished, the yellow bar on the top of the screen reports back on whether an optimal solution was found or not.

Click 'Next' to solve the model and continue.

Since the model has now been solved, we can look at the solution results. The Model overview tab lists general characteristics of the model. For example, the type of model (continuous, integer, binary), the direction of optimization (is it a minimizing model, or a maximizing model?), the numbers of variables and constraints.

If the model is two-dimensional, then the feasible region is plotted, along with the optimal point or points. So, for example, in the current model, the (unique) optimal solution satisfies x1 = 4.5, x2 = 4.5.

Click 'Next' to continue.

The Variables tab lists the variables of the model, their optimal values, and several other values. For the current model, the variables are x1 and x2. They are real-valued (as opposed to integer or binary). Both variables have optimal value 4.5.

Click 'Next' to continue.

The Constraints tab lists the constraints of the model, and the so-called slack values at the optimal solution. The current model has four constraints, named c11, c12, c13, and c14.

Click 'Next' to continue.

The Output tab lists any output generated by printf statements in the code. There are no such statements in the model, so there is nothing to display.

Click 'Next' to continue.

The Log messages tab lists any output generated by the solver. This is useful for tracing any problems in the solution process.

Click 'Next' to continue.

You have reached the end of the tour.

  • There are lots of examples in the Examples menu.
  • You can load and save models from the Model menu.
  • You can run sensitivity analysis from the Run menu.

Click 'Close' to end the tour.
